# AI 도구 도입 보안 감사 체크리스트

> 새로운 AI 도구를 조직에 도입할 때, 이 체크리스트를 순서대로 점검하세요. 한 번에 끝내는 것이 아니라 도입 전→운영 중→정기 점검으로 반복합니다.

---

## 핵심 원칙: Shift Left — 보안은 처음부터

보안을 나중에 추가하면 비용이 기하급수적으로 늘어납니다.
**도구를 선정하는 단계에서부터 보안을 검토**하세요. 도입 후 보안 문제가 발견되면 전환 비용, 데이터 이전 비용, 재교육 비용이 모두 발생합니다.

```
❌ 도구 선정 → 도입 → 사용 → 보안 문제 발견 → 대응 (비용 10배)
✅ 보안 검토 → 도구 선정 → 도입 → 사용 → 정기 점검 (비용 1배)
```

---

## 1단계: 약관 및 데이터 정책 검토

도구를 선정하기 **전에** 확인합니다.

### 데이터 학습 및 활용

- [ ] 입력 데이터가 모델 학습(training)에 사용되는가?
  - 사용됨 → **업무용 부적합** (1등급 데이터만 가능)
  - 옵트아웃 가능 → 옵트아웃 설정 확인 후 사용
  - 사용 안 됨 → 약관에 명시적으로 기재되어 있는지 확인
- [ ] 입력 데이터를 서비스 개선(improvement) 목적으로 활용하는가?
  - "학습(training)"과 "서비스 개선(improvement)"은 다른 개념 — 둘 다 확인
- [ ] 대화 내역의 보관 기간은 얼마인가?
  - 사용자가 삭제할 수 있는가?
  - 자동 삭제 기간이 설정되어 있는가?
- [ ] API 호출 시와 웹 인터페이스 사용 시 약관이 동일한가?
  - 대부분의 서비스에서 API와 웹의 데이터 정책이 다름 — 반드시 구분하여 확인

### 서버 및 법적 관할

- [ ] 데이터가 저장/처리되는 서버 위치는? (국내/해외/선택 가능)
- [ ] 해당 국가의 개인정보보호법이 한국법과 호환되는가?
- [ ] 데이터 국외 이전에 대한 동의/계약이 필요한가?
  - 개인정보보호법 제28조의8 (개인정보의 국외 이전) 해당 여부

### 보안 인증

- [ ] SOC 2 Type II 인증을 보유하고 있는가?
- [ ] ISO 27001 인증을 보유하고 있는가?
- [ ] 기타 업종별 필수 인증: HIPAA (의료), PCI DSS (금융), FedRAMP (공공) 등
- [ ] 최근 12개월 내 보안 감사 보고서를 제공할 수 있는가?

---

## 2단계: 접근 제어 및 관리 기능

도구를 **도입하기 전에** IT/보안팀과 함께 확인합니다.

### 인증 및 권한

- [ ] SSO (Single Sign-On) 연동이 가능한가? (SAML, OIDC)
- [ ] MFA (다중 인증)를 지원하는가?
- [ ] 사용자별/그룹별 접근 권한을 관리할 수 있는가?
  - 역할 기반 접근 제어 (RBAC) 지원 여부
- [ ] 퇴사자/이동자 계정을 즉시 비활성화할 수 있는가?
- [ ] 외부 공유 기능을 관리자가 제한할 수 있는가?

### 감사 및 로그

- [ ] 관리자가 사용 로그를 조회할 수 있는가?
  - 누가, 언제, 어떤 기능을 사용했는지
- [ ] 입력/출력 내용까지 감사 로그로 남기는가? (선택적)
- [ ] 로그 보관 기간은 충분한가? (최소 1년 권장)
- [ ] 로그를 SIEM 등 사내 보안 시스템으로 내보낼 수 있는가?

### API 관리

- [ ] API 키를 팀/프로젝트별로 분리 발급할 수 있는가?
- [ ] API 사용량 제한(rate limit)을 설정할 수 있는가?
- [ ] API 키 순환(rotation) 정책을 지원하는가?

---

## 3단계: 데이터 보호

도구 **도입 시** 보안팀과 함께 설정합니다.

### 전송 및 저장

- [ ] 전송 중 암호화 (TLS 1.2 이상)가 적용되는가?
- [ ] 저장 시 암호화 (AES-256 등)가 적용되는가?
- [ ] 고객 간 데이터 격리 (tenant isolation)가 보장되는가?

### 민감정보 필터링

- [ ] 입력 단계에서 민감정보를 자동 감지/차단하는 기능이 있는가?
  - DLP (Data Loss Prevention) 연동 가능 여부
- [ ] 민감정보 필터링 규칙을 커스터마이징할 수 있는가?
  - 예: 주민등록번호 패턴, 사내 프로젝트 코드명 등
- [ ] 출력에 민감정보가 포함되지 않도록 하는 가드레일이 있는가?

### 백업 및 복구

- [ ] 데이터 백업 정책이 있는가? (빈도, 보관 기간)
- [ ] 서비스 장애 시 데이터 복구 SLA는?
- [ ] 계약 종료 시 데이터 삭제 절차가 명시되어 있는가?

## 4단계: AI 특화 보안 항목

AI 도구만의 고유한 보안 리스크를 점검합니다.

### 프롬프트 인젝션 방어

- [ ] 시스템 프롬프트와 사용자 입력이 명확히 분리되는가?
- [ ] 악의적 프롬프트 주입에 대한 방어 기능이 있는가?
- [ ] 프롬프트를 통한 시스템 정보 유출 가능성을 테스트했는가?

### AI 에이전트 (자동 실행 도구)

AI가 코드 실행, 파일 접근, API 호출 등을 자동으로 수행하는 경우:

- [ ] 에이전트의 권한 범위가 **최소 권한 원칙**으로 설정되어 있는가?
- [ ] 에이전트가 접근 가능한 시스템/데이터의 목록이 정의되어 있는가?
- [ ] 고위험 작업 (삭제, 전송, 배포)은 사람의 승인이 필요한가?
- [ ] 에이전트 실행 로그가 남는가?

### 모델 출력 품질

- [ ] 할루시네이션(허위 정보 생성) 리스크에 대한 가이드라인이 있는가?
- [ ] AI 출력물의 검증 프로세스가 정의되어 있는가?
- [ ] AI 출력물임을 표시하는 정책이 있는가? (내부/외부 발송 시)

---

## 5단계: 규정 준수

법무팀/컴플라이언스팀과 함께 확인합니다.

### 개인정보보호

- [ ] 개인정보보호법 제15조 (수집·이용) 요건을 충족하는가?
- [ ] 개인정보 처리 위탁 계약이 필요한가? (제26조)
- [ ] 개인정보 영향평가 대상인가? (공공기관의 경우)
- [ ] 정보주체에게 AI 활용 사실을 고지해야 하는가?

### 산업별 규정

- [ ] 금융: 전자금융감독규정, 신용정보법 적용 여부
- [ ] 의료: 의료법, 개인정보보호법 민감정보 조항
- [ ] 공공: 국가정보보안기본지침, 클라우드 보안 인증 (CSAP)
- [ ] 제조: 산업기술보호법 적용 여부 (국가핵심기술)

### 계약 및 지적재산

- [ ] AI 생성물의 저작권/지적재산권 귀속이 명확한가?
- [ ] AI 서비스 제공자의 면책 조항 범위는 적절한가?
- [ ] SLA (서비스 수준 합의)가 업무 요구사항을 충족하는가?

## 보안 수준별 도입 방식 가이드

조직의 보안 요구 수준에 따라 도입 방식이 달라집니다.

| 구분 | SaaS Enterprise | API + 사내 프록시 | 온프레미스 |
|------|----------------|------------------|-----------|
| **보안 수준** | 기본~중 | 중~고 | 최고 |
| **데이터 통제** | 서비스 약관에 의존 | API 정책 + 프록시 제어 | 완전 자체 통제 |
| **민감정보 필터링** | 서비스 기본 기능 | 프록시에서 커스텀 필터 | 자체 구현 |
| **감사 로그** | 서비스 제공 범위 | 프록시 + API 로그 | 완전 자체 관리 |
| **적합한 조직** | 일반 기업, 스타트업 | 금융, 의료, 공공 | 방산, 국가기관, 핵심기술 |
| **비용** | 중 (라이선스) | 중~고 (인프라 추가) | 고 (서버 + 인력) |
| **도입 기간** | 1~2주 | 2~4주 | 1~3개월 |

> **판단 기준**: "우리 조직에서 가장 민감한 데이터의 등급"으로 결정하세요. 3등급(기밀) 이상의 데이터를 AI로 처리해야 한다면 온프레미스, 2등급(내부)까지라면 SaaS Enterprise 또는 API 방식이 적합합니다. [데이터 등급 가이드](data-classification.md) 참조.

## 정기 점검 일정

보안 감사는 일회성이 아닙니다. 아래 주기로 반복하세요.

| 점검 항목 | 주기 | 담당 |
|-----------|------|------|
| 약관/정책 변경 확인 | 분기 1회 | 보안팀 |
| 접근 권한 리뷰 (퇴사자, 이동자) | 월 1회 | IT팀 |
| 사용 로그 점검 (이상 사용 탐지) | 월 1회 | 보안팀 |
| 민감정보 입력 여부 샘플 점검 | 월 1회 | 보안팀 |
| 전체 보안 감사 (이 체크리스트 재점검) | 연 1회 | 보안팀 + 법무팀 |
| AI 에이전트 권한 리뷰 | 분기 1회 | IT팀 + 개발팀 |
