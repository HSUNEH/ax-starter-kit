# AI 사용 보안 가이드라인

> 조직에서 AI를 안전하게 사용하기 위한 전반적인 보안 원칙과 실천 방안입니다.

---

## 핵심 원칙 3가지

### 1. AI에 넣는 것 = 외부에 보내는 것

클라우드 AI 서비스를 사용할 때, 입력한 내용은 외부 서버로 전송됩니다.
**"이 내용을 외부 이메일로 보내도 되는가?"** 를 기준으로 판단하세요.

### 2. AI 결과를 맹신하지 않는다

AI는 확신 있게 틀릴 수 있습니다 (할루시네이션).
특히 아래 항목은 **반드시 원본과 대조**하세요:
- 수치, 통계, 재무 데이터
- 법률, 규정, 계약 조항
- 고유명사, 인용문
- 날짜, 기한

### 3. 최종 책임은 사람에게 있다

AI가 작성한 보고서, 이메일, 분석의 최종 책임은 **이를 제출/발송한 사람**에게 있습니다.
"AI가 그렇게 써줬다"는 변명이 되지 않습니다.

---

## 해야 할 것 (Do)

- ✅ AI에 입력하기 전에 [데이터 등급](data-classification.md)을 확인한다
- ✅ 개인정보는 반드시 익명화/마스킹한 후 입력한다
- ✅ AI 결과물은 반드시 검토한 후 사용한다
- ✅ Enterprise 또는 API 플랜을 사용한다 (데이터 학습 방지)
- ✅ 팀 내 AI 사용 범위를 사전에 합의한다
- ✅ 새로운 AI 도구를 도입할 때 IT/보안팀에 사전 확인한다

## 하지 말아야 할 것 (Don't)

- ❌ 고객 개인정보 (이름+연락처, 계좌번호 등)를 AI에 입력
- ❌ 비밀번호, API 키, 인증 토큰을 AI에 입력
- ❌ 미발표 재무 데이터, M&A 정보 등 기밀을 AI에 입력
- ❌ 무료/개인 플랜에 업무 데이터를 입력
- ❌ AI 결과를 검증 없이 외부에 발송
- ❌ 개인 AI 계정으로 업무 데이터를 처리

---

## AI 도구 도입 시 보안 체크리스트

새로운 AI 도구를 팀/조직에 도입할 때 확인할 사항:

### 약관 및 정책
- [ ] 입력 데이터가 모델 학습에 사용되는가? (옵트아웃 가능 여부)
- [ ] 데이터 보관 기간은? (대화 이력 삭제 가능 여부)
- [ ] 데이터가 저장되는 서버 위치는? (국내/해외)
- [ ] SOC 2, ISO 27001 등 보안 인증을 보유하고 있는가?

### 접근 제어
- [ ] SSO (Single Sign-On) 연동이 가능한가?
- [ ] 사용자별 접근 권한을 관리할 수 있는가?
- [ ] 관리자가 사용 로그를 조회할 수 있는가?

### 데이터 보호
- [ ] 전송 중 암호화 (TLS)가 적용되는가?
- [ ] 저장 시 암호화가 적용되는가?
- [ ] 고객 간 데이터 격리가 보장되는가?

## 보안 사고 대응

만약 민감한 정보를 AI에 실수로 입력했다면:

1. **즉시 해당 대화/세션을 삭제**합니다
2. **팀 보안 담당자에게 보고**합니다
3. 유출 가능성을 평가합니다:
   - Enterprise/API 플랜: 학습 미사용 보장, 리스크 낮음
   - 무료/개인 플랜: 학습에 사용될 수 있음, 리스크 높음
4. 필요 시 관련 비밀번호/API 키를 **즉시 변경**합니다
5. 개인정보 유출 시 **개인정보보호 담당자에게 즉시 보고**합니다

## 보안 환경별 AI 도입 방식

조직의 보안 요구 수준에 따라 도입 방식이 달라집니다:

| 보안 수준 | 방식 | 비용 | 적합한 조직 |
|-----------|------|------|------------|
| **기본** | SaaS Enterprise 플랜 | 중 | 일반 기업, 스타트업 |
| **강화** | API 호출 + 사내 프록시 | 중~고 | 금융, 의료, 공공 |
| **최고** | 온프레미스 (자체 서버) | 고 | 방산, 국가기관, 제조 핵심기술 |

> 💡 **팁**: 온프레미스가 반드시 최선은 아닙니다. 인건비와 운영 비용을 고려하면 Enterprise SaaS가 더 경제적인 경우가 많습니다. 보안 요구 수준에 맞는 적정 방식을 선택하세요.
