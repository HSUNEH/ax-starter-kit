# 조직용 AI 사용 정책 템플릿

> 이 문서를 복사하여 조직의 AI 사용 정책으로 사용하세요. `[괄호]` 안의 내용을 조직에 맞게 수정하면 됩니다.

---

## 사용 방법

1. 아래 전체 템플릿을 복사합니다
2. `[괄호]` 안의 내용을 조직에 맞게 수정합니다
3. 법무팀/보안팀 검토를 받습니다
4. 사내 공지 후 시행합니다

> **Dual Run 원칙**: 새 정책을 시행할 때, 기존 방식을 즉시 폐기하지 마세요. **2~4주간 기존 프로세스와 병행 운영** 한 후, 정책이 안정적으로 정착되었을 때 전환합니다. 갑작스러운 전환은 혼란과 저항을 유발합니다.

## 제1조 (목적)

이 정책은 [조직명]의 임직원이 업무에 AI 도구를 사용할 때 준수해야 할 원칙, 범위, 절차를 정하여 **업무 생산성 향상과 정보 보안을 동시에 달성**하는 것을 목적으로 합니다.

---

## 제2조 (적용 범위)

1. 이 정책은 [조직명]의 모든 임직원, 계약직, 파견직, 협력사 직원에게 적용됩니다.
2. 업무 목적으로 사용하는 모든 AI 도구에 적용됩니다:
   - 대화형 AI (ChatGPT, Claude, Gemini 등)
   - AI 코딩 도구 (GitHub Copilot, Claude Code, Cursor 등)
   - AI 기반 업무 도구 (Notion AI, Microsoft Copilot, 회의록 AI 등)
   - 자체 개발/운영하는 AI 시스템
3. 개인적 용도의 AI 사용은 이 정책의 적용을 받지 않으나, 업무 데이터를 개인 AI에 입력하는 행위는 이 정책을 따릅니다.

---

## 제3조 (승인된 AI 도구)

### 승인 도구 목록

아래 도구만 업무 목적으로 사용할 수 있습니다.

| 도구명 | 플랜 | 용도 | 사용 가능 데이터 등급 | 관리 부서 |
|--------|------|------|---------------------|-----------|
| [도구명 1] | [Enterprise] | [전사 업무] | 1~2등급 | [IT팀] |
| [도구명 2] | [API] | [개발] | 1~2등급 | [개발팀] |
| [도구명 3] | [온프레미스] | [기밀 업무] | 1~3등급 | [인프라팀] |

### 도구 승인 절차

목록에 없는 AI 도구를 사용하려면:

1. 사용자가 [IT팀/보안팀]에 도입 요청서를 제출합니다
2. [보안팀]이 [보안 감사 체크리스트](audit-checklist.md)에 따라 보안 검토를 수행합니다
3. 검토 통과 시 승인 도구 목록에 추가합니다
4. **미승인 도구에 업무 데이터를 입력하는 것은 금지**됩니다

---

## 제4조 (데이터 사용 기준)

### 데이터 등급별 사용 규칙

[데이터 등급 가이드](data-classification.md)를 기준으로 합니다.

| 등급 | 정의 | AI 사용 | 조건 |
|------|------|---------|------|
| **1등급 (공개)** | 이미 공개된 정보 | ✅ 자유 사용 | 없음 |
| **2등급 (내부)** | 사내 일반 정보 | ⚠️ 조건부 사용 | 승인된 도구 + 익명화 |
| **3등급 (기밀)** | 사업 핵심 정보 | ❌ 외부 AI 금지 | 온프레미스만 허용 |
| **4등급 (민감)** | 법적 보호 대상 | ❌ 절대 금지 | 어떤 AI에도 입력 금지 |

### 절대 금지 데이터

어떤 AI 도구에도 입력해서는 안 되는 데이터:

- 고객/직원 개인정보 (이름+연락처, 주민등록번호, 계좌번호)
- 비밀번호, API 키, 인증 토큰, 접속 정보
- 미발표 재무 데이터, M&A 정보
- [조직 특화 기밀: 예) 핵심 기술 도면, 임상 데이터 등]

### 익명화 규칙

2등급 데이터를 AI에 입력할 때는 반드시 익명화합니다:

| 원본 | 익명화 |
|------|--------|
| 김철수 과장 | 담당자 A |
| (주)ABC기업 | 거래처 X |
| 3억 2천만 원 | 약 3억 원 또는 [금액] |
| 서울시 강남구 테헤란로 123 | [주소] |
| 010-1234-5678 | [연락처] |

## 제5조 (사용자 권한 체계)

### 역할별 권한

| 역할 | 권한 | 설명 |
|------|------|------|
| **일반 사용자** | 기본 사용 | 승인된 도구로 1~2등급 데이터 처리 |
| **파워 유저** | 확장 사용 | API 직접 호출, 프롬프트 라이브러리 관리 |
| **AI 관리자** | 관리 | 도구 설정, 사용 로그 조회, 권한 관리 |
| **보안 관리자** | 감사 | 보안 감사, 정책 위반 조사, 사고 대응 |

### 권한 부여 절차

1. 기본 사용 권한은 AI 기초 교육 이수 후 자동 부여
2. 파워 유저 권한은 부서장 승인 + 추가 교육 이수
3. 관리자 권한은 [IT팀장/CISO] 승인

### 권한 회수

- 퇴사/부서 이동 시 즉시 회수 (인사 프로세스와 연동)
- 정책 위반 시 권한 일시 정지 후 재교육

---

## 제6조 (AI 출력물 관리)

### 검증 의무

1. AI가 생성한 모든 산출물은 **사용자가 내용을 검증한 후** 사용합니다
2. 아래 항목은 반드시 **원본과 대조**합니다:
   - 수치, 통계, 재무 데이터
   - 법률, 규정, 계약 조항
   - 고유명사, 인용문, 날짜
3. **최종 책임은 산출물을 제출/발송한 사람**에게 있습니다

### 라벨링

- 외부 발송 문서에 AI 생성 내용이 포함된 경우, [조직 정책에 따라 라벨링/미라벨링]
- 내부 문서는 AI 활용 여부를 표기하는 것을 권장합니다 (강제 아님)

### 저작권

- AI 생성물의 저작권은 [조직명]에 귀속됩니다
- AI 생성물을 외부에 공개할 때는 저작권 리스크를 검토합니다
- AI 도구의 약관에서 출력물 권리를 확인합니다

---

## 제7조 (AI 에이전트 사용 규칙)

AI가 자동으로 코드 실행, 파일 접근, API 호출 등을 수행하는 에이전트를 사용할 때:

### 필수 원칙

1. **최소 권한**: 에이전트에게 작업에 필요한 최소한의 권한만 부여합니다
2. **사람 승인**: 고위험 작업 (배포, 삭제, 외부 전송)은 사람이 승인해야 합니다
3. **로그 기록**: 에이전트의 모든 실행 내역을 기록합니다
4. **비용 제한**: API 호출 비용/횟수에 상한선을 설정합니다

### 환경 분리

| 환경 | 에이전트 허용 범위 |
|------|-------------------|
| 개발 (Development) | 자유 실행 허용 |
| 스테이징 (Staging) | 실행 허용, 외부 통신 제한 |
| 프로덕션 (Production) | 읽기만 허용, 변경은 사람 승인 필수 |

---

## 제8조 (교육)

### 필수 교육

| 교육 | 대상 | 시기 | 내용 |
|------|------|------|------|
| AI 기초 교육 | 전 직원 | 입사 시 / 정책 시행 시 | 정책 요약, 데이터 등급, Do/Don't |
| AI 보안 교육 | 전 직원 | 연 1회 | 보안 사고 사례, 대응 절차 |
| AI 심화 교육 | 파워 유저 | 파워 유저 신청 시 | 프롬프트 엔지니어링, API 사용법 |
| AI 관리자 교육 | 관리자 | 관리자 지정 시 | 도구 관리, 로그 분석, 사고 대응 |

### 교육 미이수 시

- AI 기초 교육 미이수 시 AI 도구 사용 권한이 부여되지 않습니다
- 보안 교육 미이수 시 기존 권한이 일시 정지됩니다

---

## 제9조 (위반 시 조치)

| 위반 수준 | 예시 | 조치 |
|-----------|------|------|
| **경미** | 미승인 도구에 1등급 데이터 입력 | 구두 주의 + 재교육 |
| **보통** | 2등급 데이터 익명화 미실시 | 서면 경고 + 권한 일시 정지 + 재교육 |
| **중대** | 3~4등급 데이터 AI 입력 | 징계 + 권한 회수 + 보안 사고 대응 절차 진행 |
| **심각** | 고의적 대량 데이터 유출 | 징계 + 법적 조치 검토 |

> **참고**: 이 정책의 목적은 처벌이 아니라 **안전한 AI 활용 문화 정착**입니다. 실수로 인한 첫 번째 위반은 교육 기회로 활용합니다.

---

## 제10조 (정책 운영)

### 정책 관리

- **담당 부서**: [IT팀 / 보안팀 / 디지털혁신팀]
- **문의 채널**: [이메일 / 슬랙 채널 / 사내 포털]
- **정책 검토 주기**: 6개월 (AI 기술 변화 속도를 고려)
- **개정 절차**: 담당 부서 초안 → 법무팀 검토 → [승인권자] 승인 → 전사 공지

### 도입 일정 (Dual Run)

새 정책 도입 시 기존 방식과 병행하여 단계적으로 전환합니다.

| 기간 | 단계 | 내용 |
|------|------|------|
| 1~2주차 | **안내 기간** | 정책 공지 + 교육, 위반 시 안내만 (제재 없음) |
| 3~4주차 | **병행 운영** | 정책 적용 시작, 경미한 위반은 안내 + 재교육 |
| 5주차~ | **정식 운영** | 정책 전면 적용, 위반 시 제9조에 따라 조치 |

## 부칙

### 부칙 1: 업종별 추가 규정

조직의 업종에 따라 아래 항목을 추가하세요.

**금융업**
- AI를 이용한 투자 자문/추천은 관련 법규 준수 필요
- 고객 신용정보는 어떤 AI에도 입력 금지 (신용정보법)
- 전자금융감독규정에 따른 외부 클라우드 사용 승인 필요

**의료업**
- 환자 의료 기록은 어떤 AI에도 입력 금지
- AI 진단 보조 도구는 의료기기 인증 확인 필요
- 임상 데이터는 비식별화 후에도 IRB 승인 필요

**공공기관**
- 국가정보보안기본지침 준수
- CSAP (클라우드 보안 인증) 획득 서비스만 사용
- 국가핵심기술 관련 데이터는 AI 사용 절대 금지

**제조업**
- 핵심 기술 도면, 공정 데이터는 3등급 이상으로 분류
- 협력사와의 AI 사용 범위를 계약에 명시
- 산업기술보호법 적용 대상 확인
