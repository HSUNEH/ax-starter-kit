# 데이터 등급별 AI 사용 가이드

> 어떤 데이터를 AI에 넣어도 되는지, 등급별로 판단하는 기준입니다.

---

## 데이터 등급 분류

| 등급 | 정의 | 예시 | AI 사용 |
|------|------|------|---------|
| **1등급 (공개)** | 이미 공개된 정보, 유출되어도 영향 없음 | 공개 보도자료, 공식 홈페이지 내용, 일반 업계 지식 | ✅ 자유롭게 사용 |
| **2등급 (내부)** | 사내에서 일반적으로 공유되는 정보 | 사내 공지, 일반 업무 문서, 회의 안건 | ⚠️ 조건부 사용 |
| **3등급 (기밀)** | 유출 시 사업에 직접적 영향 | 재무 데이터, 미발표 전략, 계약 조건, 기술 설계 | ❌ 사용 금지 (또는 온프레미스만) |
| **4등급 (민감)** | 법적 보호 대상 | 개인정보 (이름+연락처), 의료/금융 정보, 비밀번호 | ❌ 절대 금지 |

---

## 등급별 사용 규칙

### 1등급 (공개) — ✅ 자유 사용
- 어떤 AI 도구든 사용 가능
- 특별한 조치 불필요

### 2등급 (내부) — ⚠️ 조건부 사용
- **Enterprise 계약** 체결된 AI 도구만 사용 (학습에 데이터가 사용되지 않는 플랜)
- API 호출 방식 권장 (웹 채팅보다 안전)
- 민감 정보(이름, 금액 등)는 익명화 후 사용
  - 예: "김철수 과장" → "담당자 A"
  - 예: "3억 2천만 원" → "약 3억 원" 또는 "[금액]"

### 3등급 (기밀) — ❌ 외부 AI 금지
- 외부 클라우드 AI 서비스 사용 금지
- **온프레미스 AI** (사내 서버에 설치된 모델)만 허용
- 온프레미스 환경이 없으면 AI 사용 불가 — 기존 방식으로 처리

### 4등급 (민감) — ❌ 절대 금지
- 어떤 AI 도구에도 입력 금지 (온프레미스 포함)
- 개인정보보호법, 정보통신망법 등 법적 제재 대상

---

## 판단이 어려울 때

```
이 데이터를 AI에 넣어도 될까?
→ "이 데이터가 경쟁사에 유출되면?"을 상상해보세요.

- 별일 없다 → 1등급
- 좀 불편하다 → 2등급 (익명화 후 사용)
- 큰일난다 → 3등급 (온프레미스 또는 사용 금지)
- 법적 문제 → 4등급 (절대 금지)
```

**확실하지 않으면, 한 단계 높은 등급으로 처리하세요.**

## AI 도구별 보안 수준

| 보안 수준 | 설명 | 사용 가능 등급 | 예시 |
|-----------|------|---------------|------|
| **무료/개인 플랜** | 입력 데이터가 학습에 사용될 수 있음 | 1등급만 | ChatGPT Free, Gemini Free |
| **유료/Pro 플랜** | 학습 미사용 옵트아웃 가능, 단 약관 확인 필요 | 1~2등급 | ChatGPT Plus, Claude Pro |
| **Enterprise 플랜** | 데이터 격리, 학습 미사용 보장, 감사 로그 | 1~2등급 | ChatGPT Enterprise, Claude Enterprise |
| **API 호출** | 데이터 학습 미사용 (대부분의 API 약관), 로그 관리 가능 | 1~2등급 | OpenAI API, Anthropic API |
| **온프레미스** | 사내 서버에서 구동, 외부 전송 없음 | 1~3등급 | Qwen, GLM, Llama (자체 서버) |

## 실전 체크리스트

AI에 데이터를 입력하기 전, 이 체크리스트를 확인하세요:

- [ ] 이 데이터의 등급은 몇 등급인가?
- [ ] 사용하려는 AI 도구의 보안 수준은 해당 등급을 허용하는가?
- [ ] 개인정보 (이름, 연락처, 주민번호 등)가 포함되어 있지 않은가?
- [ ] 익명화/마스킹이 필요한 부분은 처리했는가?
- [ ] 결과물에 원본 데이터가 그대로 노출되지 않는가?

## 팀 보안 정책 템플릿

아래를 복사하여 팀 보안 정책으로 사용하세요:

```
[팀명] AI 사용 보안 정책

1. 승인된 AI 도구: [도구명, 플랜]
2. 사용 가능 데이터 등급: 1등급, 2등급 (익명화 필수)
3. 금지 데이터: 고객 개인정보, 재무 데이터, 미발표 전략
4. 익명화 규칙: 실명 → 직책/코드, 금액 → 범위, 날짜 → 유지
5. 문의: [보안 담당자/채널]

최종 수정: [날짜]
```
