# AX 성과 지표(KPI) 정의 프레임워크

> AI 전환의 성과를 "좋은 것 같다"가 아닌 **수치로 증명**하기 위한 프레임워크입니다. 측정하지 않으면 개선할 수 없고, 증명하지 않으면 확산할 수 없습니다.

---

## 핵심 원칙

### 1. 팀 평균이 아닌 개인별로 측정한다

"팀 AI 활용률 70%"는 의미 없는 숫자입니다. 10명 중 7명이 주 1회 쓴 건지, 3명이 매일 쓴 건지 알 수 없기 때문입니다.

**개인별 기여도를 측정**해야 정확한 성과 파악과 맞춤 지원이 가능합니다.

```
❌ "우리 팀 AI 활용률 70%" → 누가 쓰고 누가 안 쓰는지 모름
✅ "10명 중 7명이 주 3회 이상 사용, 3명은 미사용" → 미사용자 케어 가능
```

### 2. 절감된 시간의 행방을 추적한다

AI로 시간을 절약했다면, 그 시간은 어디로 갔는가?

| 시간의 행방 | 가치 | 측정 방법 |
|------------|------|-----------|
| **고부가가치 업무**에 재투자 | 높음 | 업무 시간 비율 변화 추적 |
| **학습/역량 개발**에 사용 | 높음 | 교육 이수 시간, 신규 스킬 |
| **추가 업무 처리** | 중간 | 처리 건수 변화 |
| **그냥 사라짐** (측정 안 함) | 없음 | — |

> 절감 시간의 일부를 **공식적으로 학습에 할당**하세요. "AI로 1시간 줄었으니 1시간 더 일해"가 아니라, "30분은 학습에 쓰세요"가 지속 가능한 모델입니다.

### 3. 비용은 인건비로 환산한다

경영진은 "시간 절감"보다 **"비용 절감"**에 반응합니다. 절감된 시간을 인건비로 환산하세요.

```
월 절감 비용 = 절감 시간(시간) × 시급 × 대상 인원

예시:
- 대상: 마케팅팀 8명
- 절감: 주당 1인 3시간 × 4주 = 월 12시간/인
- 시급: 40,000원 (연봉 5,000만 원 기준)
- 월 절감 비용: 12시간 × 40,000원 × 8명 = 3,840,000원
- 연 절감 비용: 약 4,600만 원
- AI 도구 비용: 월 $20 × 8명 = 약 21만 원/월, 약 250만 원/년
- 순 절감: 약 4,350만 원/년
```

## 정량 지표

### 핵심 지표 (필수 측정)

| 지표 | 정의 | 측정 방법 | 목표 예시 |
|------|------|-----------|-----------|
| **업무 처리 시간** | AI 적용 대상 업무의 평균 소요 시간 | Before/After 비교 (타이머 또는 자기 보고) | 30~50% 감소 |
| **AI 활용률** | 주 1회 이상 AI를 업무에 사용하는 직원 비율 | 주간 체크인 또는 사용 로그 | 4주차 80% 이상 |
| **비용 절감액** | 절감 시간 × 시급 × 인원 | 위 환산 공식 적용 | AI 도구 비용의 3배 이상 |

### 보조 지표 (권장 측정)

| 지표 | 정의 | 측정 방법 | 목표 예시 |
|------|------|-----------|-----------|
| **산출물 품질** | 수정 횟수, 피드백 건수 변화 | Before/After 비교 | 수정 횟수 20% 감소 |
| **처리 건수** | 동일 시간 내 처리한 업무 건수 | 업무 시스템 로그 | 20~30% 증가 |
| **AI 도구 사용 빈도** | 1인당 주간 AI 사용 횟수 | 사용 로그 또는 자기 보고 | 주 5회 이상 |
| **프롬프트 재사용률** | 팀 프롬프트 라이브러리 활용 비율 | 라이브러리 사용 로그 | 50% 이상 |

### 자동 측정이 가능한 지표

AI 도구의 관리자 대시보드 또는 API로 자동 수집할 수 있는 지표:

| 지표 | 데이터 소스 | 활용 |
|------|------------|------|
| **토큰 사용량** | AI 서비스 관리자 대시보드 / API 로그 | 실제 사용량 추적, 비용 관리 |
| **활성 사용자 수** | AI 서비스 관리자 대시보드 | 활용률 자동 산출 |
| **사용 시간대/패턴** | API 로그 | 업무 통합 수준 파악 |
| **대화 수/세션 수** | AI 서비스 관리자 대시보드 | 사용 빈도 자동 산출 |

> **토큰 사용량 기반 측정의 장점**: 자기 보고에 의존하지 않아 정확하고, 별도 작업 없이 자동 수집됩니다. Enterprise 플랜이나 API 사용 시 대부분 제공됩니다.

## 정성 지표

### 필수 수집

| 지표 | 측정 방법 | 시기 |
|------|-----------|------|
| **직원 만족도** | 5점 척도 설문 | 도입 전 / 2주차 / 4주차 |
| **가장 도움이 된 업무** | 주관식 응답 | 4주차 |
| **가장 불편한 점** | 주관식 응답 | 2주차 / 4주차 |
| **AI 도입 전후 업무 방식 변화** | 주관식 응답 | 4주차 |

### 수집 팁

- 설문은 **짧게** (5분 이내 완료 가능하게)
- 주관식은 1~2문항만
- 익명 설문으로 솔직한 응답 유도
- [설문 템플릿](survey-templates.md)을 그대로 사용하세요

## 직군별 KPI 가이드

### 사무직 (기획, 마케팅, 인사, 재무)

| KPI | 측정 방법 | 기대 효과 |
|-----|-----------|-----------|
| 보고서 작성 시간 | Before/After 타이머 | 40~60% 감소 |
| 이메일 응답 시간 | 이메일 시스템 로그 | 30~50% 감소 |
| 회의록 정리 시간 | Before/After 비교 | 70~80% 감소 |
| 데이터 분석 시간 | Before/After 비교 | 30~50% 감소 |

**측정 포인트**: 반복적 텍스트 작업의 시간 절감이 핵심

### 개발직

| KPI | 측정 방법 | 기대 효과 |
|-----|-----------|-----------|
| 코드 작성 시간 | PR 생성 주기, 커밋 빈도 | 30~50% 감소 |
| 코드 리뷰 시간 | 리뷰 완료 시간 | 20~40% 감소 |
| 버그 수정 시간 | 이슈 Open→Close 시간 | 20~30% 감소 |
| 문서화 완성도 | 문서 커버리지 | 50% 이상 증가 |

**측정 포인트**: 개발 속도보다 **개발 품질** (버그율, 리뷰 품질)에 더 주목

### 영업/CS

| KPI | 측정 방법 | 기대 효과 |
|-----|-----------|-----------|
| 제안서 작성 시간 | Before/After 비교 | 40~60% 감소 |
| 고객 응답 시간 | CS 시스템 로그 | 30~50% 감소 |
| FAQ 자동 처리율 | 챗봇/AI 자동 응답 비율 | 30% 이상 |
| 고객 만족도 | CSAT 점수 | 유지 또는 향상 |

**측정 포인트**: 응답 속도 향상이 고객 만족으로 이어지는지 확인

## 측정 일정

| 시점 | 수집 항목 | 담당 |
|------|-----------|------|
| **도입 전 (Baseline)** | Before 업무 시간, 도입 전 설문 | AX 담당자 |
| **1주차** | AI 활용률 (첫 사용 여부) | 챔피언 |
| **2주차** | 주간 체크인 설문, 사용 빈도 | AX 담당자 |
| **3주차** | 주간 체크인 설문, 중간 점검 | AX 담당자 |
| **4주차** | After 업무 시간, 최종 설문, 비용 환산 | AX 담당자 |
| **이후 월 1회** | 활용률, 사용 빈도, 월간 절감 비용 | AX 담당자 |

## KPI 대시보드 구성 예시

경영진/리더에게 보고할 때 아래 구조로 정리하세요.

```
┌─────────────────────────────────────────────────┐
│  AX 성과 대시보드 — [팀명] [기간]                    │
├─────────────────────────────────────────────────┤
│                                                 │
│  [핵심 지표]                                      │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐        │
│  │ 시간 절감  │ │ 활용률    │ │ 비용 절감  │        │
│  │ -42%     │ │ 85%     │ │ 380만원/월│        │
│  └──────────┘ └──────────┘ └──────────┘        │
│                                                 │
│  [업무별 시간 절감]                                 │
│  보고서 작성  ████████░░ -45%                      │
│  이메일 답변  ██████░░░░ -35%                      │
│  회의록 정리  █████████░ -80%                      │
│  데이터 분석  ██████░░░░ -30%                      │
│                                                 │
│  [개인별 활용 현황]                                 │
│  주 3회 이상: 6명 | 주 1~2회: 2명 | 미사용: 2명      │
│                                                 │
│  [절감 시간 활용 내역]                               │
│  고부가가치 업무: 40% | 추가 업무: 35% | 학습: 25%   │
│                                                 │
└─────────────────────────────────────────────────┘
```

> 실제 대시보드는 노션, 구글 시트, 또는 사내 BI 도구로 구현하세요. 위 구조를 그대로 옮기면 됩니다.

## 흔한 측정 실수

| 실수 | 문제 | 해결 |
|------|------|------|
| **Before를 안 재고 시작** | After만 있으면 "얼마나 좋아졌는지" 증명 불가 | 도입 전 반드시 Baseline 측정 |
| **팀 평균만 보고** | 소수 파워유저가 평균을 끌어올림, 실제 보급 현황 왜곡 | 개인별 분포로 확인 |
| **시간만 측정** | 경영진이 공감 못 함 | 비용(인건비)으로 환산 |
| **자기 보고에만 의존** | 과대/과소 보고 발생 | 자동 측정 데이터와 교차 검증 |
| **1회 측정 후 끝** | 지속 효과를 증명 못 함 | 월 1회 정기 측정 |

## 성과 기반 보상 가이드

### 직군별 차등 보상 기준

AI 활용 성과는 직군마다 기대치가 다릅니다. 동일한 잣대를 적용하지 마세요.

| 직군 | 핵심 성과 기준 | 보상 방식 예시 |
|------|---------------|---------------|
| **사무직** | 절감 시간, 산출물 품질 | 우수 활용 사례 시상, 학습 시간 추가 부여 |
| **개발직** | 코드 품질, 개발 속도 | 신기술 교육 지원, 컨퍼런스 참가 |
| **영업/CS** | 고객 응답 속도, 처리 건수 | 인센티브 연계, 자동화 성과 공유 |
| **관리직** | 팀 활용률, 워크플로우 개선 건수 | 리더십 성과에 반영 |

### 보상 원칙

- **금전 보상보다 시간 보상이 효과적**: "AI로 절감된 시간의 일부를 자유롭게 쓸 수 있다"
- **팀 보상과 개인 보상을 병행**: 팀 성과는 공유, 개인 기여는 인정
- **처벌이 아닌 동기부여**: AI 미사용자를 처벌하지 말고, 사용자를 보상

*이 프레임워크는 4주 시범 운영 기준으로 설계되었습니다. 조직 규모와 목표에 맞게 지표와 기준을 조정하세요. [성과 리포트 템플릿](report-template.md)으로 결과를 정리하고, [설문 템플릿](survey-templates.md)으로 정성 데이터를 수집하세요.*
