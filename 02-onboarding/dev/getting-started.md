# 개발자를 위한 AI 도구 시작 가이드

> 개발 경험이 있지만 AI 코딩 도구는 처음인 분을 위한 가이드입니다. 30분 안에 첫 세팅을 마치고, 바로 업무에 적용할 수 있습니다.

---

## Step 1: AI 코딩 도구 선택하기

팀에서 이미 도구를 정했다면 이 단계를 건너뛰세요.

### 도구 비교

| 도구 | 방식 | 강점 | 약점 | 추천 상황 |
|------|------|------|------|-----------|
| **GitHub Copilot** | IDE 인라인 자동완성 | 코드 작성 중 실시간 제안, IDE 통합 최고 | 복잡한 맥락 이해 제한적 | 코드 작성량이 많은 일상 개발 |
| **Claude Code** | CLI 에이전트 | 프로젝트 전체 맥락 이해, 자동 실행 | CLI 환경 적응 필요 | 리팩토링, 멀티파일 작업, 하네스 구축 |
| **Cursor** | AI 네이티브 에디터 | VS Code 기반, 자연어 편집 | 별도 에디터 설치 필요 | VS Code 사용자, 빠른 시작 |
| **Windsurf** | AI 네이티브 에디터 | 에이전트 모드 강력 | 생태계 작음 | 에이전트 방식 선호 시 |

### 어떤 도구를 골라야 할까?

```
Q1. 기존 IDE를 유지하고 싶은가?
├── Yes → Copilot (인라인 자동완성)
└── No
    Q2. CLI를 선호하는가?
    ├── Yes → Claude Code (CLI 에이전트)
    └── No → Cursor 또는 Windsurf (AI 에디터)
```

> **팁**: 도구를 하나만 써야 하는 건 아닙니다. **Copilot(자동완성) + Claude Code(에이전트)** 조합이 효과적입니다. 일상적 코딩은 Copilot, 복잡한 작업은 Claude Code로 분담하세요.

## Step 2: 설치 및 세팅

### GitHub Copilot

```bash
# 1. GitHub Copilot 구독 (Individual $10/월 또는 조직 라이선스)
# 2. IDE 확장 설치
# VS Code: Extensions에서 "GitHub Copilot" 검색 → Install
# JetBrains: Settings → Plugins → "GitHub Copilot" 검색 → Install

# 3. GitHub 계정 로그인
# IDE에서 팝업 → GitHub OAuth 로그인

# 4. 동작 확인: 아무 파일에서 주석 작성 후 자동완성 제안이 뜨는지 확인
```

**핵심 단축키:**

| 동작 | Mac | Windows/Linux |
|------|-----|---------------|
| 제안 수락 | `Tab` | `Tab` |
| 다음 제안 | `Option + ]` | `Alt + ]` |
| 인라인 채팅 | `Cmd + I` | `Ctrl + I` |
| 채팅 패널 | `Cmd + Shift + I` | `Ctrl + Shift + I` |

### Claude Code

```bash
# 1. 설치 (Node.js 18+ 필요)
npm install -g @anthropic-ai/claude-code

# 2. 인증
claude
# 첫 실행 시 Anthropic 계정 로그인 또는 API 키 입력

# 3. 프로젝트에서 시작
cd your-project
claude

# 4. 동작 확인: "이 프로젝트의 구조를 설명해줘"로 테스트
```

**핵심 명령어:**

| 명령어 | 설명 |
|--------|------|
| `claude` | 대화형 모드 시작 |
| `claude "질문"` | 한 번만 질문하고 종료 |
| `claude commit` | 변경 사항 커밋 |
| `/help` | 도움말 |
| `Esc` | 현재 작업 중단 |

### Cursor

```bash
# 1. cursor.com에서 다운로드 및 설치
# 2. VS Code 설정 자동 임포트 (선택)
# 3. Cursor Settings → Models에서 사용할 모델 선택
# 4. 동작 확인: Cmd+K (인라인 편집) 또는 Cmd+L (채팅)
```

**핵심 단축키:**

| 동작 | Mac | Windows/Linux |
|------|-----|---------------|
| 인라인 편집 | `Cmd + K` | `Ctrl + K` |
| 채팅 | `Cmd + L` | `Ctrl + L` |
| Composer | `Cmd + I` | `Ctrl + I` |
| 파일 참조 | `@파일명` | `@파일명` |

## Step 3: 첫 번째 활용 — 코드 이해

새로운 코드베이스에 합류했거나, 레거시 코드를 파악해야 할 때 가장 효과적입니다.

### 프로젝트 구조 파악

```
이 프로젝트의 전체 구조를 설명해줘.
- 주요 디렉토리별 역할
- 핵심 진입점 (엔트리포인트)
- 주요 의존성
```

### 특정 코드 이해

```
src/auth/middleware.ts 파일을 분석해줘.
- 이 미들웨어의 역할
- 인증 흐름 (요청 → 응답까지)
- 다른 모듈과의 의존 관계
```

### 에러 분석

```
이 에러 메시지를 분석해줘:

[에러 메시지 붙여넣기]

- 원인이 무엇인지
- 어떤 파일에서 수정해야 하는지
- 수정 방법
```

> **팁**: AI에게 코드를 설명하게 하면, 본인이 직접 읽는 것보다 빠르게 전체 맥락을 파악할 수 있습니다. 특히 레거시 코드, 다른 팀의 코드, 오픈소스 라이브러리 분석에 유용합니다.

## Step 4: 습관 만들기

### 첫째 주: "하루 1번 AI에게 물어보기"

- 코드 리뷰 전에 AI로 먼저 확인
- 에러 메시지를 AI에 붙여넣기
- 결과가 별로여도 괜찮습니다 — **쓰는 것 자체가 목표**

### 둘째 주: "주요 업무 1개를 AI로 전환"

- 반복적인 작업을 선택 (테스트 작성, 문서화, 보일러플레이트 등)
- Before/After 시간을 기록

### 셋째 주 이후: "AI가 기본 도구가 된 상태"

- 프롬프트를 팀과 공유
- [프롬프트 엔지니어링 가이드](prompt-engineering.md)로 품질 향상
- [워크플로우 템플릿](workflow-templates.md)으로 업무 프로세스에 통합

## 보안 체크

AI 코딩 도구를 사용하기 전에 반드시 확인하세요.

- [ ] 팀에서 승인된 도구를 사용하고 있는가?
- [ ] Enterprise/API 플랜인가? (개인 무료 플랜은 데이터 학습에 사용될 수 있음)

## 자주 묻는 질문

**Q: AI가 작성한 코드를 그대로 커밋해도 되나요?**
> 안 됩니다. AI가 생성한 코드는 반드시 **직접 리뷰한 뒤** 커밋하세요. 보안 취약점, 불필요한 의존성, 라이선스 문제가 있을 수 있습니다. 최종 책임은 커밋한 개발자에게 있습니다.

**Q: AI가 우리 코드베이스를 학습하나요?**
> 도구와 플랜에 따라 다릅니다. Enterprise 플랜은 대부분 학습에 사용하지 않습니다. 반드시 서비스 약관을 확인하세요. 자세한 내용은 [보안 감사 체크리스트](../../03-security/audit-checklist.md)를 참조.

**Q: Copilot과 Claude Code, 둘 다 써야 하나요?**
> 필수는 아닙니다. 하나만 써도 충분합니다. 다만 역할이 다릅니다: Copilot은 타이핑하면서 실시간 자동완성, Claude Code는 복잡한 작업을 대화로 처리. 둘 다 쓰면 각각의 강점을 활용할 수 있습니다.

**Q: AI로 대체되는 건 아닌가요?**
> AI는 개발자를 대체하는 것이 아니라, 개발자의 생산성을 높이는 도구입니다. AI를 잘 활용하는 개발자와 그렇지 않은 개발자의 생산성 차이가 벌어지고 있습니다. 도구를 빨리 익히는 것이 경쟁력입니다.
